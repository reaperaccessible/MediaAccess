MediaAccess - Journal des modifications

Version 2.48

Nouveau : lecture des livres (EPUB et DAISY texte seul) à voix haute avec une voix neuronale Microsoft Edge, sans aucune installation. Dans l'onglet Livres des options, cochez « Lire les livres avec une voix neuronale (Edge, en ligne) », puis choisissez la langue, la voix (avec aperçu) et le débit. MediaAccess narre les paragraphes avec une vraie voix de synthèse, en pré-synthétisant le paragraphe suivant pour enchaîner sans blanc. La voix système (SAPI) reste le réglage par défaut ; toutes les commandes (lecture, pause, arrêt, navigation par paragraphe / chapitre / page, signets, reprise de position) fonctionnent à l'identique dans les deux modes. Repli automatique sur la voix système en cas de problème réseau. Contribution de math65.

Nouveau : les notes « Quoi de neuf » affichées dans la fenêtre de mise à jour s'affichent désormais dans votre langue (français ou anglais), comme le journal des modifications (F2) et le manuel (F1). Auparavant, ces notes étaient toujours affichées en français.

Version 2.46

Une nouvelle action « Ouvrir le journal des modifications » (Outils → Actions, catégorie Principale) permet d'y assigner le raccourci de votre choix.

Version 2.45

Amélioration (accessibilité) : le mode de répétition est désormais un sous-menu Lecture → Répéter avec trois choix en boutons radio — Répétition désactivée / Répéter la piste / Tout répéter — dont l'option active est annoncée par le lecteur d'écran. On peut ainsi vérifier le mode courant et choisir directement « Répétition désactivée », sans devoir faire défiler les modes. La touche E continue de faire défiler les trois modes. Signalé par un utilisateur.

Version 2.44

Nouveau : lecture des sous-titres à voix haute avec une voix neuronale Microsoft Edge, sans aucune installation. Dans l'option « Énoncer les sous-titres » (onglet Parole), choisissez la méthode « voix neuronale » : MediaAccess lit les répliques avec une vraie voix de synthèse, en pré-synthétisant à l'avance pour rester synchronisé, et baisse légèrement le son de la vidéo pendant la voix. Source des sous-titres : fichier .srt/.vtt voisin, sinon la piste embarquée active (extraite automatiquement). Options : filtre de langue, aperçu de voix, niveau de baisse du volume, débit. Repli automatique sur le lecteur d'écran en cas de problème réseau, et message clair pour les sous-titres en image (non lisibles). Contribution de math65.

Version 1.92

Fix a single-instance race condition that split a multi-file Explorer selection across two MediaAccess windows. When you selected, say, 8 files of mixed extensions (.mkv, .mp4, .m4a, .mov, .mid, etc.) in Windows Explorer and pressed Enter, Explorer launched several MediaAccess.exe processes in parallel. The first to grab the single-instance mutex won, but its main window hadn't been created yet (BASS init, settings load, keymap parsing take a few hundred milliseconds). Version 1.76

Several improvements to video file playback, driven by user reports on TNT TV recordings and Captvty replay files.

The first bug only triggered when the "Load folder" option in Preferences was enabled. Picking a single video file through File > Open then left MediaAccess with an empty playlist and nothing playing, because the folder-expansion routine was filtering its scan to audio-only extensions — the video file the user had just picked was silently dropped on the way to the playlist. The second bug affected .mp4 files that contain BOTH video and audio (typical Captvty TV replay downloads).
